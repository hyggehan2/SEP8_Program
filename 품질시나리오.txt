품질 우선 순위표(QAW 결과)
품질속성: 성능
속성 상세화: 실시간 가격조회 정보 최신화
시나리오: 사용자가 시스템과 상호작용을 하는 동안 올바른 정보를 실시간으로 제공한다.
우선순위-중요성:H
우선순위-난이도:H

품질속성: 상호운용성
속성 상세화: API의 실시간 가격조회/거래 정보 최신화
시나리오: API를 통해 서버에서 필요한 가격조회/거래 정보를 실시간으로 받는다.
우선순위-중요성:H
우선순위-난이도:M

품질속성: 보안성
속성 상세화: 시스템의 정보보안
시나리오: 악의적인 봇을 활용하여 사용자 계정에 접근하려고 할때 접근을 막아야한다.
우선순위-중요성:H
우선순위-난이도:M

사용편의성 시나리오
사용자는 시스템의 가격조회 활동 동작을 취소하기를 원한다.
이러한 취소 동작은 1초내에 이루어져야한다.
자극의 근원: 사용자
자극: 오류의 영향 최소화
대상: 시스템
환경: 작동상태
응답: 현재 작동을 중단하고자 함
응답측정: 취소가 1초내에 이루어져야함

성능 시나리오
가격 조회에서 서버는 확률적으로 분당 500개의 가격 조회를 데이터베이스에 요청하고,
가격정보의 표시는 평균 1초의 대기시간 내에 처리된다.
자극의 근원: 서버
자극: 가격 조회
대상: 데이터베이스
환경: 정상적인 작동상태
응답: 가격 조회
응답측정: 대기시간 평균 1초

상호운용성
시스템은 API를 통해 필요한 정보를 확률적으로 분당 500개를 요청한다.
API를 통해 요청해서 받은 정보는 데이터베이스에 최신화하는데 평균 대기시간 1초 내에 처리된다.
자극의 근원: 시스템
자극: 정보 요청
대상: API
환경: 정상적인 작동상태
응답: API를 통해 정보 요청/데이터베이스에 저장
응답측정: 대기시간 평균 2초

신뢰성 시나리오
시스템이 서버 관리자의 통제없이 작동하도록 한다.
서버는 사용자가 요청하는 가격조회와 거래를 계속해서 수행한다.
자극의 근원: 사용자
자극: 가격조회/거래 요청
대상: 시스템
환경: 정상적인 작동상태
응답: 가격조회/거래 수행
응답측정: 8시간동안 관리자의 통제 없이 작동하게 하고, 시스템의 오류로 중단되는 경우를 확인한다.

보안성 시나리오
시스템에 악의적인 봇을 활용한 공격을 통해 로그인을 시도한다.
시스템은 이를 구분하여 로그인을 시도한 계정을 중지시켜 접근을 5분간 막는다.
자극의 근원: 악의적인 봇
자극: 로그인 시도
대상: 시스템
환경: 로그인 요청상태
응답: 로그인을 시도한 계정에 대한 접근 거부
응답측정: 5분간 로그인 시도 거부

유지보수성 시나리오
사용자의 개인정보 체계를 바꾸려고한다.
사용자의 정보는 ID/PW만 검증하는 단계에서 지갑을 추가하여 원화/개수를 포함하도록 하려고 한다.
자극의 근원: 관리자
자극: 데이터베이스와 메소드 업데이트
대상: 시스템
환경: 일시중지전 정상적으로 작동상태
응답: 업데이트된 내용 적용
응답측정: 관리자가 원하는 내용을 잘 적용하는지 기능검증을 통해 확인

이식성 시나리오
가상화폐 시세 통합 시스템을 운영하는 서버를 옮기고자 한다.
구버전의 운영체제에서 작동하고자 한다.
자극의 근원: 서버 관리자/사용자
자극: 서버 이동/구버전 사용
대상: 시스템
환경: 새로운 위치로 옮겨진 작동초기상태/구버전 운영체제에서 작동상태
응답: 작동시작
응답측정: 시스템을 작동시키고 잘 작동하는지 확인
